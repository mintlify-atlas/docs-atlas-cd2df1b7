This guide walks you through generating your first API client using agent mode — the default mode where the AI navigates the browser autonomously on your behalf.

<Note>
  Before starting, make sure you have completed the [installation steps](/installation), including setting your `ANTHROPIC_API_KEY`.
</Note>

<Steps>
  <Step title="Launch the CLI">
    Start the interactive CLI from any directory:

    ```bash
    reverse-api-engineer
    ```

    You'll see the banner and a prompt showing the current mode:

    ```
    [agent] >
    ```

    The tool starts in **agent** mode by default. The mode label in the prompt reflects the active mode.

    <Tip>
      Press **Shift+Tab** to cycle through the four available modes: `agent` → `manual` → `engineer` → `collector`. The prompt label updates in real time.
    </Tip>
  </Step>

  <Step title="Enter a task description">
    Type a plain-English description of what you want to capture. For example, to fetch job listings from Apple's careers page:

    ```
    [agent] > fetch all apple jobs from their careers page
    ```

    In agent mode, you don't need to provide a URL — the AI navigates to the right place on its own.
  </Step>

  <Step title="Wait for the agent to finish">
    The agent opens a browser, navigates to the site, and interacts with it to trigger the API calls you described. You'll see real-time progress in the terminal as it works.

    When the agent is done, it closes the browser and hands the captured traffic to Claude for analysis.
  </Step>

  <Step title="Review the generated files">
    Once generation is complete, your API client is saved to two locations:

    - **Permanent storage**: `~/.reverse-api/runs/scripts/{run_id}/`
    - **Local copy**: `./scripts/apple_jobs_api/` (a descriptive name derived from your task)

    The local `./scripts/` directory contains three files:

    ```
    ./scripts/apple_jobs_api/
    ├── api_client.py      # The generated API client
    ├── example_usage.py   # Ready-to-run usage examples
    └── README.md          # Endpoint documentation
    ```

    Open `example_usage.py` to see how to call the generated client right away.
  </Step>
</Steps>

## The four modes

Press **Shift+Tab** at the prompt to switch modes. The current mode is always shown in the prompt prefix.

<CardGroup cols={2}>
  <Card title="agent (default)" icon="robot">
    The AI navigates the browser autonomously. Describe your goal and the agent handles browsing, clicking, and capturing network traffic on its own.
  </Card>
  <Card title="manual" icon="hand-pointer">
    You drive the browser while the tool records traffic. Useful for sites that require human judgment, login flows, or complex interactions the agent can't handle reliably.
  </Card>
  <Card title="engineer" icon="wrench">
    Re-run AI code generation against a previous capture. Use this to iterate on an existing session without re-browsing the site. Enter a run ID or use `@id <run_id>` tags.
  </Card>
  <Card title="collector" icon="database">
    AI-powered data collection that exports to JSON and CSV. Describe the data you want in natural language; the agent searches and structures it automatically.
  </Card>
</CardGroup>

## Example: Apple jobs API

Here's what the full workflow looks like end to end:

```bash
$ reverse-api-engineer
[agent] > fetch all apple jobs from their careers page

  autonomous agent will navigate on its own

# Agent opens browser, navigates to apple.com/careers, interacts with the page
# Claude analyzes the captured HAR file and generates code

# Scripts saved to: ./scripts/apple_jobs_api/
```

The generated `api_client.py` includes:

- Clean function interfaces for each discovered endpoint
- Authentication handling (cookies, headers, tokens)
- Type hints and docstrings throughout
- Error handling for common failure cases

```python ./scripts/apple_jobs_api/example_usage.py
from api_client import AppleJobsClient

client = AppleJobsClient()
jobs = client.get_jobs(location="United States", team="Software Engineering")
for job in jobs:
    print(f"{job['title']} — {job['location']}")
```

## Slash commands

While inside the interactive CLI, these commands are available at any prompt:

| Command | Description |
|---------|-------------|
| `/settings` | Configure model, SDK, agent provider, and output options |
| `/history` | View past runs with timestamps and costs |
| `/messages <run_id>` | Inspect full LLM conversation logs for a run |
| `/help` | Show all available commands |
| `/exit` | Quit the CLI |
